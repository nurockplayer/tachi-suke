# Phase 1DN OpenSearch Output Guard Plan

## Goal

Add a narrow build-output guard for `opensearch.xml` so browser/search-adjacent discovery stays aligned with TachiSuke's reviewed static search contract.

## Success Criteria

- Built `opensearch.xml` is checked exactly by `pnpm check:seo`.
- The OpenSearch description keeps the reviewed namespace, site name, description, UTF-8 input encoding, and one HTML search template.
- The search template uses the configured site origin and `/en/search?q={searchTerms}`.
- Production/configured-site checks do not ship the fallback example domain.
- The output does not point at account placeholders, locale search variants, search index JSON, private data, hosted search, or backend routes.
- No runtime feature, route, UI, auth, database, form, Worker, Function, or deployment behavior changes are introduced.

## Current Context

- `/opensearch.xml` is generated by `src/pages/opensearch.xml.ts`.
- Public pages already link it through `BaseLayout`.
- Existing SEO checks verify basic OpenSearch presence and template shape, but not the full reviewed XML contract.

## Constraints

- Use pnpm only.
- Keep Phase 1 static-first.
- Do not add dependencies.
- Use `rtk` only for git and gh commands.

## Worker Routing

- `controller_fallback_reason=trivial_self_only`
- This is a small single-output test extension in an already inspected generated artifact.
- Worker delegation would add overhead without reducing risk for this exact output contract check.
